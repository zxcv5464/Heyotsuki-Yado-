# 菜單管理測試清單

## 權限

- [ ] `owner` 可以進入 `admin/menu.html` 並編輯菜單。
- [ ] `admin` 可以進入 `admin/menu.html` 並編輯菜單。
- [ ] `staff` 進入時顯示「此帳號沒有菜單管理權限」。
- [ ] `staff` 看不到菜單、分類與品項表單。
- [ ] 使用 `staff` session 直接呼叫寫入 API 時，Supabase RLS 仍會拒絕。

## 菜單切換與基本資料

- [ ] 可以切換「湯宿菜單」，載入 `menu_key = menu`。
- [ ] 可以切換「喫茶菜單」，載入 `menu_key = menu2`。
- [ ] 後台沒有新增第三份菜單或修改 `menu_key` 的介面。
- [ ] 可以修改 `title`、`short_title`、`english_title`、`description`、`is_visible` 與 `sort_order`。
- [ ] `href` 僅供檢視，無法從 UI 修改。
- [ ] 儲存後重新載入目前菜單資料。
- [ ] Supabase 讀取失敗時顯示錯誤訊息，而不是整頁空白。

## 分類

- [ ] 新增一個測試分類，前台對應菜單會顯示。
- [ ] 編輯分類名稱、副標題、備註、排版類型、顯示狀態與排序。
- [ ] 新分類未填 `sort_order` 時使用目前最大值加 10。
- [ ] 分類依 `sort_order` 由小到大排列。
- [ ] 隱藏分類後，前台不顯示，後台仍看得到並標示「已隱藏」。
- [ ] 分類可選擇 `detailed`、`compact`、`nested_detailed`、`nested_compact`。
- [ ] `nested_detailed` 會顯示在排序中上一個一般分類底下。
- [ ] `nested_compact` 會顯示在排序中上一個一般分類底下。
- [ ] 畫面沒有永久刪除分類按鈕。

## 品項

- [ ] 選擇分類後只顯示該分類的品項。
- [ ] 新增一個測試品項，前台對應分類會顯示。
- [ ] 新品項未填 `sort_order` 時使用同分類最大值加 10。
- [ ] 編輯品項名稱、描述、價格、強調狀態、顯示狀態與排序。
- [ ] 價格可以輸入 `+100,000 Gil` 等文字格式。
- [ ] 隱藏品項後，前台不顯示，後台仍看得到並標示「已隱藏」。
- [ ] 修改價格後，前台價格會更新。
- [ ] 調整 `sort_order` 後，前台排序會更新。
- [ ] 勾選 `featured` 後，前台顯示原有強調樣式。
- [ ] 畫面沒有永久刪除品項按鈕。

## 前台與安全

- [ ] `menu.html` 正常讀取 `menu`。
- [ ] `menu2.html` 正常讀取 `menu2`。
- [ ] 所有分類都隱藏時，前台不會回退顯示本地分類。
- [ ] 某分類所有品項都隱藏時，前台不會回退顯示本地品項。
- [ ] title、notice、label、name、description、price 中的 HTML 不會執行。
- [ ] textarea 換行會顯示為換行。
- [ ] 舊資料中的 `<br>` 仍可顯示為換行。
- [ ] 舊品項名稱 HTML 只呈現純文字，不保留可執行標籤。
- [ ] 菜單的桌機與手機視覺維持原本樣式。

## 手機版

- [ ] 可切換兩份菜單。
- [ ] 可新增及編輯分類。
- [ ] 可選擇分類並新增及編輯品項。
- [ ] 所有表單與列表沒有水平溢位。
